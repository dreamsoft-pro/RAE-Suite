# RAE-Suite: The Deterministic Agentic Factory
## Manifesto and Architecture Specification v1.3 (The Adaptive Iron Protocol)

### 1. Vision & Core Paradigm
RAE-Suite is a **Deterministic Operating System for Agentic Engineering**. It transforms the uncertainty of Large Language Models into high-density, auditable, and industrial-grade software artifacts. 

The suite follows the **"Architecture as a Service" (RaaS/CaaS)** model, governed by an **Adaptive Consensus Mechanism** to ensure both mathematical correctness and production-grade execution speed.

---

### 2. The Adaptive Council of Elders (Dynamic Governance)
Instead of rigidly querying 9 models for every task, RAE-Suite utilizes **Adaptive Consensus (1-to-3 Scaling)**. The number of models invoked per layer scales dynamically based on the risk level and confidence score of the operation.

*   **Fast-Path (Low Risk):** Uses only Model A.
*   **Consensus-Path (High Risk / Low Confidence):** Invokes Models A, B, and C, requiring a 2/3 majority vote.

| Layer | Type | Default Models (A / B / C) | Responsibility |
| :--- | :--- | :--- | :--- |
| **L1: Working** | Production | **Gemini 2.5 Flash / DeepSeek V3 / Qwen 2.5** | High-speed code generation, unit testing. |
| **L2: Expert** | Auditing | **Claude 3.5 Sonnet / Gemini 1.5 Pro / GPT-4o** | Structural review, contract generation. |
| **L3: Supreme** | Final Verdict | **Claude 4.5 / Gemini 2.0 Pro / GPT-o1** | Behavioral symmetry, ISO compliance. |

---

### 3. The 6-Stage Production Funnel (The Standard)

#### Stage 1: Ontological Ingestion (Discovery)
*   **Action:** Reconstructs the intent field. *Szubar Mode* is activated only if the semantic gap is too large.

#### Stage 2: Behavior-Driven Contracting (Phenix)
*   **Action:** Generation of `.contract.yml` (The Law).
*   **Adaptive Check:** L3 Supreme Council approves the contract *only* if the Time Complexity or Security boundaries are altered. Otherwise, L2 Expert Council handles it.

#### Stage 3: Swarm Execution (Hive Swarm)
*   **Mechanism:** L1 Worker Swarm implements the code.
*   **Verification:** Writer-Auditor feedback loop.

#### Stage 4: Hard Frames & Deterministic Validation
*   **Action:** Physical execution of tests and security scanners. 
*   **Anti-Loop Mechanism:** If the swarm fails to resolve warnings after 3 iterations, the *Zero-Warning Policy* forces an escalation to the L2 Expert Council instead of getting stuck in an endless loop.

#### Stage 5: Observability & Telemetry Injection (OTel)
*   **Action:** Automatic injection of OTel instrumentation.
*   **Result:** Real-time tracking of code performance.

#### Stage 6: ISO 42001 Governance & Meta-Reflection
*   **Action:** L3 Supreme Council signs the Audit Report. RAE-Core updates Multi-Armed Bandit weights.
